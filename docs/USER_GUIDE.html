📚 Key Concepts

Tokens & Token Types

A token is roughly 4 characters of text. Claude splits all text into tokens for processing.

Token Type Description Cost Example
Input Your messages + system prompts + context. Sent TO Claude. $3.00/1M
Output Claude's responses. Generated BY Claude. $15.00/1M (usually 5x more expensive)
Cache Read Tokens from the prompt cache that Claude reused (doesn't process them again). $0.30/1M (90% cheaper!)
Cache Create Tokens added to the cache for future reuse. Small one-time cost. $3.75/1M (25% input cost)
💡 Example: You send a 1M token prompt.
  • First run: Cost = 1M × $3.00 input = $3.00
  • Add to 1h cache: + 1M × $0.75 cache create = $3.75 total
  • Second run (reusing cache): 0.1M new input + 0.9M cache read = 0.1M × $3.00 + 0.9M × $0.30 = $0.57 (saves $3.18!)

Cache Hit Rate

Percentage of tokens retrieved from the prompt cache vs. processed fresh.

Cache Hit Rate = Cache Read Tokens / (Cache Read Tokens + Input Tokens) × 100%
Interpretation:
  • 0%: No caching used. Each session processes everything from scratch.
  • 50%: Half your tokens come from cache, half are fresh. Good reuse!
  • 90%+: Excellent. You're reusing a lot of context (e.g., large codebases).

Net Cache Savings

How much you save by using prompt caching vs. processing everything fresh.

Gross Saved = Cache Read × (Input Rate - Cache Read Rate)
Extra Paid = Cache Create × (Cache Create Rate - Input Rate)
Net Saved = Gross Saved - Extra Paid
Interpretation:
  • Positive = Caching saved you money (good!)
  • Negative = Caching cost more (you're not reusing enough)

Cost per Message vs. Cost per Token

Cost per message ($/msg): Total cost of a response ÷ number of responses.

Cost per token: Calculated differently for input, output, and cache.

Use $/msg to:
  • Compare efficiency across different Claude models
  • Benchmark: "On average, each Claude response costs $X"

Session vs. Message

  • Session: An entire Claude Code conversation from start to finish. One session = many messages.
  • Message: One turn in the conversation. You send a message → Claude responds → that's a message exchange.

Tool Calls

When Claude uses tools (file reads, edits, bash commands, API calls, etc.), each invocation is a "tool call".

Tool calls show up in metrics as:
  • Top tools chart: Which tools are used most
  • Cost breakdown: Not separately, but they affect token count (planning + execution)

Timezone and Daily Buckets

All daily metrics ("Today", daily breakdown table) are bucketed based on your configured timezone, not UTC.

Why this matters:
  • If you work 9am-5pm in Tokyo (JST), set timezone to "Asia/Tokyo"
  • "Today" metrics will show 9am Tokyo time as the start, not midnight UTC
  • Makes budgeting and daily tracking align with your actual work day

Model Fallback and Pricing Hierarchy

If a Claude model isn't in your pricing table, Token Pulse falls back to the longest matching prefix.

Model: "claude-opus-4-7-20250615" → No exact match?
→ Try "claude-opus-4-7" → No match?
→ Try "claude-opus-4" → Use this pricing
Use this to:
  • Set pricing once per model family (e.g., just "claude-opus-4")
  • New Claude versions automatically inherit the family pricing
  • Override specific versions if needed (add "claude-opus-4-7-20250615")

🎯 Tips & Tricks for Optimizing Costs

1. Use Prompt Caching

When: You reuse the same large context (e.g., a codebase, documentation, system prompt).
Benefit: 90% cost reduction on repeated context (cache read at $0.30/1M vs. input at $3.00/1M).
How to measure: Check the "Cache" card on Overview. High hit rate = good!

2. Choose the Right Model

Check: Models table on Overview. Compare $/msg across models.
Consideration: Cheaper models (Sonnet) might require longer prompts; expensive models (Opus) might solve faster. Measure end-to-end cost, not just $/msg.

3. Monitor Daily Budget with Projections

Check: Projection card on Overview.
If "Projected month" > budget: Something changed. Investigate the daily chart for spikes.

4. Analyze Expensive Sessions

Check: Sessions page sorted by cost (descending).
For each expensive session: Open it and review the "Cost per response" chart. Which responses cost the most? Why?

5. Watch Output Tokens**

Remember: Output tokens cost 5x more than input.
Optimization: If Claude is verbose, ask for "brief" or "concise" responses. Check output tokens in cost breakdown.
